About This Project

This personal website was built as part of the CSE216 practical Git project. The goal was not just to create a website, but to demonstrate a real Git workflow — including feature branching, merge conflicts, safe undo operations, reflog recovery, and semantic release tagging.

Every commit in this project tells a story of how the site evolved, from a blank HTML file to a styled, multi-page website with proper version control practices.

What I Learned

Repository Setup

Initialized a Git repo, configured identity, and created essential files — README, .gitignore, and CHANGELOG.

Branching & Merging

Created feature branches for docs, UI, and refactoring. Learned the difference between fast-forward and merge commits.

Conflict Resolution

Deliberately created a merge conflict by editing the same line on two branches, then resolved it by combining both versions.

Undo & Recovery

Used git revert for safe undo and git reflog to recover a commit lost after a hard reset.

Release Tagging

Tagged the final version as v1.0.0 with a matching CHANGELOG entry — proper semantic versioning.
